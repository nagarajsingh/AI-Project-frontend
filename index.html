Azure DevOps Pipeline Trigger

Upload documents for automated data extraction

Send PDFs, Word documents, and spreadsheets to the backend service so it can extract the required metadata and trigger your Azure DevOps pipeline.

Accepted: PDF, Word, Excel, PowerPoint, or text files (max 25 MB).

Use the HTTPS endpoint that accepts multipart/form-data.

Ready to upload.

What happens next?

  1. The backend service parses the document and extracts structured data.
  2. Validation rules are applied to confirm required metadata.
  3. The Azure DevOps pipeline is triggered with the extracted payload.

Need to support additional document types? Add them to the input accept list and confirm your backend can parse them.
